# Codex 交接记录

日期：2026-05-21

## 项目目标

项目继续命名为「涨停狙击手」。目标是从旧项目重构为一个可持续运行的 A 股 AI 量化分析与模拟交易系统：

- 自动抓取 24 小时新闻
- 使用 DeepSeek `deepseek-v4-flash` 进行新闻情绪、事件类型、行业、个股、利好利空程度分析
- 多 Agent 分析：新闻情绪、事件影响、技术走势、风控
- 同步必盈行情，交易日开盘期间同步分时，非交易时段停止行情和自动交易
- 结合新闻与 K 线做关联分析，输出短期上涨/下跌评分
- 生成每日买入信号、卖出建议、持仓、历史成交、交割单
- 模拟账户持续买卖，并用收益反馈参数
- 后台支持调度、日志、参数、接口、邮件、部署运维

## 当前关键实现

- 前台已拆成桌面工作台和独立移动端 App Shell。
- 移动端五个入口职责不重复：
  - `概览`：账户总览、今日动作摘要、系统运行日志
  - `买入`：唯一显示买入信号
  - `持仓`：当前仓位、分时估值、T+1冻结、盈亏
  - `成交`：历史成交、交割单、历史计划收益
  - `新闻`：新闻与结构化事件，不混入买入候选
- 后端账户估值优先使用当日分时最新价；无分时再回退日 K。
- 自动交易循环限制在 A 股交易时段内执行。
- 邮件通知、调度日志、安全扫描、部署文档已经进入项目结构。

## 重要口径

- 盈亏口径：总盈亏 = 已实现盈亏 + 持仓按最新分时估值的浮动盈亏。
- 买入费用计入持仓成本，刚买入的持仓可能因费用显示小幅浮亏。
- A 股 T+1：当天买入显示冻结，次日才进入可用数量。
- 昨日买入出现在今天持仓是正常跨日结转，不是今日重复买入。
- 自动交易不应在休市时段运行；手动触发可以保留。

## 安全规则

- 不提交 `.env`。
- 不提交 `backend/data/config.json`。
- 不提交 `backend/data/admin_credentials.json`。
- 不提交 `backend/data/ws_token_secret.txt`。
- 不提交真实历史运行数据、日志、缓存，除非明确决定做私有仓库数据迁移。
- 上传 GitHub 前运行：

```powershell

```
